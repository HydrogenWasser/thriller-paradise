# 游戏设计方案

## 游戏设计



## 一、 视觉风格与美术设计

- **媒介**：纯文本流（Pure Text Stream）。
- **核心理念**：排版即视觉。利用纯黑背景、极端的留白、富文本删除线（Strikethrough）、文本颜色突变以及打字机节奏的失控，来营造认知被篡改的不适感。
- **UI 布局**：
  - **顶部栏**：状态面板（剧本、难度、惊吓值、理智状态）。
  - **感官区**：用一两句极其短促、带有强烈心理暗示的文字描述当前角色的**第一直觉**（例如：*“后颈传来一阵凉意”*）。
  - **叙事区**：详尽的剧本描述与互动反馈。
  - **指令区**：纯命令交互。

## 二、 核心系统逻辑

### 2.1 惊吓值系统 (Terror Points)

惊吓值是恶人格为了掩盖真相而发动的精神防御机制。随着惊吓值升高，文本呈现出不同程度的“病变”：

- **50% 乱码（认知模糊）**：关键信息被系统屏蔽。例如：`你看到床上的女人是 [数据删除]，她正看着你。` 玩家强制要求查看会导致惊吓值飙升。
- **80% 系统撒谎（篡改记忆）**：恶人格接管系统，故意提供错误的“安全选项”诱导玩家走向 Bad End。例如，系统用高亮文本命令玩家去交一份不存在的报告，玩家若听信则惊吓值拉满；若输入指令 `检查挂历日期`，则揭穿谎言。
- **90% 心跳失控（精神崩溃边缘）**：文本滚动忽快忽慢，甚至选项会像受惊的虫子一样在屏幕上乱跑。玩家必须在极度干扰下，稳稳输入核心自我认知指令（如 `我不是平田`）来镇压狂暴。

### 2.2 脑补力与双层现实 (Reasoning & Dual Reality)

- **设计目的**：不提供视觉引导，玩家必须通过仔细阅读文字中的矛盾点来破局。
- **玩法循环**：玩家通过积累隐藏线索（如第一幕的钥匙），在遭遇后续逻辑漏洞时（如第四幕莫名多出的钥匙），主动在 `[ 指令输入 ]` 框进行质问。
- **现实覆写 (Reality Override)**：一旦输入正确的破局指令，当前的表象（Illusion）文本会被系统用红色删除线强行划掉，并追加极其冰冷、写实的客观文本（Reality）。惊吓值随之大幅下降，理智值上升。

## 三、 游戏程序开发方案

### 3.1 引擎与场景节点架构 (Scene Tree)

- **引擎版本**：Godot 4.x (基于 Godot 4 的 RichTextLabel 富文本系统实现删除线和动画特效)。
- **Main (Node2D)** - 挂载主控脚本 `GameMaster.gd`
  - **LogicManager (Node)** - 负责 JSON 剧本解析与状态机
  - **AudioManager (Node)** - 负责背景底噪与 SFX
  - **CanvasLayer (UI)**
    - **Background (ColorRect)** - 纯黑 #000000
    - **MainLayout (VBoxContainer)** - 占满全屏，边距 20px
      - **Header, SenseBox, StoryLog, InputArea** (结构保持不变，`StoryLog` 必须开启 BBCode 支持)。
    - **CRT_Overlay (ColorRect)** - 覆盖全屏，挂载 CRT Shader，提供扫描线、色散和微弱的屏幕弯曲。

### 3.2 剧本系统增强 (Extended JSON Schema)

在原有的选项驱动基础上，**新增 `hidden_breaks` 数组**，用于支撑双层现实的骇入玩法。



### 3.3 关键功能逻辑实现 (Logic Specs)

- **指令解析器 (Command Parser)**：
  - 优先检测玩家输入是否命中当前节点的 `hidden_breaks[].keywords`。
  - 若命中且满足前置线索（`requires_flag`），则打断正常流程，触发 `break_text` 并执行现实跳转。
  - 若未命中且不是基础选项（1, 2, 3），则作为“失败/瞎猜”处理，惊吓值上升，输出乱码警告。
- **动态打字机效果**：使用 Tween 或逐字符增加可见字符，遇到标点符号增加 0.2s 停顿。
- **认知干扰 (Glitch System)**：当惊吓值变动时，触发 CRT Shader 的 shaking 参数造成屏幕微抖。惊吓值 > 60 时，随机将 5% 的输出字符替换为乱码。

## 游戏程序开发方案

### 1. 项目概述

- **引擎版本：** Godot 4.x
- **游戏类型：** 纯文字交互 / 心理恐怖 / 命令行模拟
- **核心风格：** 仿终端 UI、极简主义、基于《惊悚乐园》剧本逻辑。
- **视觉规范：** 纯黑背景、等宽字体、BBCode 染色、CRT 着色器干扰。
- **工程目录**： D:\Projects\ThrillerParadise\thriller-paradise
- **Godot编辑器路径**：D:\Godot_v4.6.1-stable_win64.exe\Godot_v4.6.1-stable_win64.exe
- **Godot Console路径**：D:\Godot_v4.6.1-stable_win64.exe\Godot_v4.6.1-stable_win64_console.exe

### 2. 场景节点架构 (Scene Tree)

请按以下结构组织 Main.tscn：

- **Main (Node2D)** - 挂载主控脚本 GameMaster.gd
  - **LogicManager (Node)** - 负责 JSON 剧本解析与状态机
  - **AudioManager (Node)** - 负责背景底噪与 SFX
  - **CanvasLayer (UI)**
    - **Background (ColorRect)** - 纯黑 #000000
    - **MainLayout (VBoxContainer)** - 占满全屏，边距 20px
      - **Header (RichTextLabel)** - 显示：[剧本] [难度] [惊吓值] [状态]
      - **Divider_Top (HSeparator)**

        - **Prefix (Label)** - 显示 [ 指令输入 ] >
        - **CommandInput (LineEdit)** - 扁平化风格，无边框，自动获取焦点
    - **CRT_Overlay (ColorRect)** - 覆盖全屏，挂载 CRT.gdshader

### 3. 核心数据结构 (Data Structure)

剧本文件 hirata_world.json 示例结构：

```
{
  "office_drawer": {
    "chapter": "act4",
    "sense": "最下面的抽屉被锁死了。",
    "description": "你试着拉其他抽屉。最下面那个纹丝不动，似乎锁得很死。\n\n你摸了摸西装口袋，里面竟然躺着三把冰凉的金属钥匙。你掏出其中一把，准备试试能不能捅开抽屉的锁。",
    "options": {
      "用钥匙开锁": "office_unlock"
    },
    "hidden_breaks": [
      {
        "intent": "question_keys",
        "keywords": ["钥匙", "哪来的", "消失", "第三把", "不对", "数量", "车锁"],
        "requires_flag": "street_examine_keys_done",
        "break_text": "\n[color=red]【系统警告：检测到逻辑悖论，理智协议强制覆写】[/color]\n\n[s]你掏出其中一把，准备试试能不能捅开抽屉的锁。[/s]\n\n一阵锥心刺骨的头痛袭来。你突然意识到这根本不合理。台风摧毁了你的家，你住在连玻璃都没有的废旧长屋，为什么你身上会带着三把做工精密的钥匙？\n\n那根本不是这间办公室的钥匙，那是...你昨天夜里，强行从某个带血的制服口袋里拽出来的！",
        "next_node": "office_memory_flashback",
        "terror_change": -20,
        "reasoning_change": 15
      }
    ]
  }
}
```

| **新增扩展字段**       | **类型**        | **说明**                                                     |
| ---------------------- | --------------- | ------------------------------------------------------------ |
| **`hidden_breaks`**    | `Array`         | 隐藏的“现实覆写”触发器列表。不显示在常规选项中，仅在后台监听玩家的 `[ 指令输入 ]`。 |
| **`intent`**           | `String`        | 该隐藏触发器的意图标识，用于程序内部辨识与调试（如 `question_keys`）。 |
| **`keywords`**         | `Array<String>` | 触发该逻辑的关键词库。玩家输入的指令中只要包含其中任意组合（根据引擎分词逻辑），即可尝试触发。 |
| **`requires_flag`**    | `String`        | **核心推理锁**。玩家必须在之前的剧本中触发过特定事件（例如在 Act 1 检查过口袋获得了 `street_examine_keys_done` 标记），该隐藏打破机制才会生效，防止玩家背板盲猜。 |
| **`break_text`**       | `String`        | 覆写发生时强行插入的文本。必须包含富文本标签（如 Godot 的 `[color=red]` 和 `[s]` 删除线），用于在视觉上呈现“幻觉被涂抹、真实被揭露”的割裂感。 |
| **`next_node`**        | `String`        | 打破幻觉后，强制跳转到的“真实世界”剧本节点 ID。              |
| **`terror_change`**    | `Int`           | 触发覆写后，惊吓值的变化量（通常为负数，代表恐慌被理智压制）。 |
| **`reasoning_change`** | `Int`           | 触发覆写后，理智值/脑补力的变化量（通常为正数）。            |
